📚 지식 습득 및 기능 실습

기초 이론: 확률과 경우의 수

확률의 정의

확률은 미래에 발생할 사건에 대한 믿음을 0과 1 사이의 값으로 측정한 것입니다. 결과를 예측하기 어려운 '랜덤(RANDOM)'한 상황에서, 사건에 대한 상대 빈도(relative frequency) 정보를 통해 예측의 정확도를 높일 수 있습니다.

경우의 수: 순열과 조합

순열 (Permutation): n개의 서로 다른 원소들에서 r개를 뽑아 '순서대로' 정렬하는 경우의 수입니다. (예: 4자리 비밀번호 찾기)
조합 (Combination): n개의 서로 다른 원소들에서 '순서에 상관없이' r개를 뽑는 경우의 수입니다. (예: 남학생 15명 중 청소당번 2명 뽑기)

파이썬 Random 모듈 탐구

난수(Random Number)란?

주사위를 던질 때 어떤 숫자가 나올지 아무도 예상하지 못하는 것처럼, 무작위로 생성되는 수를 난수라고 합니다. 컴퓨터는 복잡한 계산을 통해 사람이 볼 때 마치 임의의 값인 것처럼 보이는 '의사 난수(Pseudo-random)'를 생성합니다. 파이썬에서는 `random` 모듈을 통해 다양한 난수 관련 기능을 제공합니다.

주요 랜덤 함수 종류

함수 기능 및 설명
random.randint(a, b) a와 b 사이의 정수(a, b 포함) 중 하나를 무작위로 반환합니다.
random.choice(리스트) 리스트의 요소 중 하나를 무작위로 선택하여 반환합니다.
random.shuffle(리스트) 리스트의 요소 순서를 무작위로 섞습니다. (원본 리스트가 변경됩니다)
random.sample(리스트, k) 리스트의 요소 중 k개를 중복 없이 뽑아 새로운 리스트로 반환합니다.
random.uniform(a, b) a와 b 사이의 실수(소수) 중 하나를 무작위로 반환합니다. 
# random 모듈의 다양한 함수들

import random

# 0.0과 1.0 사이의 실수(float) 난수 생성
print("random.random():", random.random())

# 지정된 범위(1~6)의 정수(int) 난수 생성
print("random.randint(1, 6):", random.randint(1, 6))

# 리스트의 순서를 무작위로 섞음 (원본 리스트 변경)
my_list = ['A', 'B', 'C', 'D']
random.shuffle(my_list)
print("random.shuffle(['A','B','C','D']):", my_list)

# 리스트의 요소 중 하나를 무작위로 선택
print("random.choice(['A','B','C','D']):", random.choice(['A', 'B', 'C', 'D']))

# 리스트의 요소 중 지정된 개수(k=2)만큼 중복 없이 뽑아 새로운 리스트 생성
print("random.sample(['A','B','C','D'], k=2):", random.sample(['A', 'B', 'C', 'D'], k=2))

import random

# 주사위를 굴릴 횟수를 1000번으로 설정
num_rolls = 1000

# 각 눈이 나온 횟수를 저장할 리스트 (0으로 초기화)
counts = [0] * 6  # counts[0]은 1의 눈, counts[1]은 2의 눈 ...

for _ in range(num_rolls):
    roll = random.randint(1, 6)
    counts[roll - 1] += 1 # 리스트 인덱스는 0부터 시작하므로 1을 빼줌

print(f"--- {num_rolls}번 주사위를 굴린 결과 ---")
for i, count in enumerate(counts):
    print(f"숫자 {i+1} 나온 횟수: {count} 번")

                    

                         
                    

                    

                

            

            

                실습 2: 자리 배치 (중복 확인 로직)
                 

                     
원리 (첨부파일 p.5)

                     
무작위로 자리를 뽑되, 이미 배정된 자리인지 `check` 리스트를 통해 확인합니다. `check` 리스트의 해당 인덱스 값이 0(False)이면 빈자리이므로 학생을 배정하고, 1(True)로 바꿔 다시 선택되지 않도록 합니다. 이 과정을 모든 학생이 배정될 때까지 반복합니다.

                     
import random

# 학생 명단 (PDF의 깨진 텍스트를 정리)
n = ['지아', '이태', '형주', '지석', '동은', '홍규', '한국', '나나', '현웅', '지훈']
num_students = len(n)

# 최종 자리 리스트 (-1로 초기화)
seat = [''] * num_students
# 자리가 배정되었는지 확인하는 리스트 (0으로 초기화)
check = [0] * num_students

i = 0
while i < num_students:
    t = random.randint(0, num_students - 1)
    # 아직 배정되지 않은 자리라면 (check[t]가 0이라면)
    if check[t] == 0:
        check[t] = 1  # 배정 완료 표시 (1로 변경)
        seat[t] = n[i] # t번 자리에 i번째 학생을 배정 (원본 코드 수정)
        i = i + 1

print("--- 자리 배치 결과 ---")
# 2줄로 출력하기 (예: 5명씩)
for i in range(num_students):
    print(seat[i], end="\\t")
    if (i + 1) % 5 == 0:
        print()

                     

                        
                    

                     

                

            

            

                실습 3: 자리 배치 (seed와 shuffle 활용)
                

                     
원리 (첨부파일 p.6)

                     
이 코드는 훨씬 효율적인 방법입니다. `random.shuffle()` 함수는 리스트의 순서를 직접 무작위로 섞어주므로, 중복을 걱정할 필요가 없습니다. 여기에 `random.seed()`를 결합하여 누가 실행해도 동일한 결과를 얻을 수 있는 공정한 프로그램을 만듭니다.

                     
Seed 구체화: 컴퓨터는 결정론적으로 작동하기 때문에 스스로 '진짜' 무작위를 만들 수 없습니다. 대신, 복잡한 수학 공식을 사용하여 무작위처럼 보이는 숫자들의 나열(의사 난수)을 만듭니다. `seed` 값은 이 공식의 '시작점'이 되는 숫자입니다. 동일한 `seed` 값을 사용하면 항상 동일한 순서의 난수들이 생성되므로, 결과를 재현하거나 모두에게 동일한 결과를 보여주어 공정성을 증명할 때 매우 중요하게 사용됩니다.

                     

                        
                        
동일한 Seed는 항상 동일한 순서의 결과를 보장합니다!

                     

                     
import random

# 학생 명단
n = ['지아', '이태', '형주', '지석', '동은', '홍규', '한국', '나나', '현웅', '지훈']

# 사용자로부터 시드값 입력받기
# try-except: 사용자가 숫자가 아닌 값을 입력하거나 취소할 경우를 대비한 오류 처리
try:
    # input() 함수로 사용자에게 입력을 요청
    s_input = input("시드값을 입력하세요 (숫자만): ")
    # 사용자가 입력을 취소하면 s_input은 None이 됩니다.
    if s_input is None:
        # 사용자가 취소했음을 알리고 기본값 사용
        raise TypeError("입력이 취소되었습니다.")
    # 입력받은 문자열을 정수(int)로 변환
    s = int(s_input)
# 사용자가 숫자가 아닌 것을 입력(ValueError)하거나 취소(TypeError)하면
except (ValueError, TypeError) as e:
    # 안내 메시지를 출력하고 기본값(42)을 사용
    print(f"안내: {e}. 기본 시드값 42를 사용합니다.")
    s = 42

# 위에서 결정된 숫자를 난수 생성기의 시드(seed)로 설정
random.seed(s)

# 리스트의 순서를 무작위로 섞음. seed가 같으면 항상 같은 순서로 섞임
random.shuffle(n)

# shuffle은 원본 리스트를 직접 바꾸므로, 결과를 seat 변수에 복사하여 저장
seat = n[:]

print("\\n--- 자리 배치 결과 ---")
# 2줄로 출력하기 (예: 5명씩)
for i in range(len(seat)):
    # 한 명씩 출력하되, 줄바꿈 없이 한 칸 띄움
    print(seat[i], end="\\t")
    # 만약 현재 출력한 학생이 5번째, 10번째... 학생이라면 줄을 바꿈
    if (i + 1) % 5 == 0:
        print()

                     

                        
                    

                     

                

            

             

                심화: 시간 복잡도와 효율성
                

                    
시간 복잡도(Time Complexity)란? (첨부파일 p.6)

                    
문제를 해결하는 데 입력값의 크기에 따라 처리 시간이 얼마나 걸리는지를 나타내는 척도입니다. 데이터의 양이 적을 때는 차이를 못 느끼지만, 수만, 수백만 건의 데이터를 처리할 때는 알고리즘의 효율성이 매우 중요해집니다. 통상적으로 짧은 시간에 문제를 해결하는 알고리즘이 좋은 알고리즘입니다.

Big-O 표기법

                     
알고리즘의 성능(시간 복잡도)을 수학적으로 표기하는 방법입니다.

                     
    
                         
  • O(1): 입력 데이터 크기와 상관없이 언제나 일정한 시간이 걸리는 알고리즘.
    • 
                         
  • O(n): 입력 데이터 크기에 비례해서 처리 시간이 걸리는 알고리즘. (예: 리스트 전체 탐색)
    • 
                         
  • O(n²): 입력 데이터 크기의 제곱에 비례해서 처리 시간이 걸리는 알고리즘. (예: 이중 for문)
    • 
                     

                

            

            

                [번외] 정보 소양 쌓기: 군 부대 배치는 어떻게 결정될까?
                

                    
seed 값의 실제 활용 사례

                    
군 입대 후 훈련소에서 받게 되는 부대 배치는 전산 프로그램을 통해 공정하게 진행됩니다. 이 과정에서 바로 우리가 배운 `seed` 값의 원리가 중요하게 사용됩니다.

                    

                        <공개 전산 분류 프로그램에 의한 신병 부대 분류 절차>

                        신병 부대 분류는 전산 프로그램에 22개의 난수가 입력되어 무작위로 이루어진다. 22개 난수는 부대 분류 당시의 '연, 월, 일, 시, 분, 초(14자리)'와 장정 대표 4인, 가족 대표 4인이 임의로 정한 숫자로 한다.
                    

                    
즉, 예측 불가능한 현재 시간과 여러 사람이 직접 참여하여 만든 숫자를 조합하여 `seed` 값을 만듭니다. 이를 통해 누구도 결과를 예측하거나 조작할 수 없는 공정한 무작위 배정 시스템을 만드는 것입니다.

                     

                        
                        
                    

                

            

        


        

            
🌟 최종 프로젝트: 조건 기반 조 추첨 프로그램

            

                [STEP 3] 최종 완성 코드 보기
                

                    
원리

                    
이 프로그램은 성별과 같은 특정 조건을 고려하여 조를 추첨하는 최종 버전입니다. 학생 명단을 외부 파일에서 읽어오는 것처럼 시뮬레이션하고, 학생들의 참여(seed 값 입력)를 통해 공정성을 확보하며, 복잡한 조건(여학생 수 배정 등)을 처리하는 알고리즘을 포함합니다.

                    
참고: 원본 코드는 `.txt` 파일에서 명단을 읽어오지만, 웹 환경에서는 직접 실행할 수 있도록 코드 내에 학생 명단을 미리 정의했습니다.

                    
import random

# --- 웹 실행을 위한 데이터 시뮬레이션 ---
# 실제로는 boy.txt, girl.txt 파일에서 명단을 읽어와야 합니다.
boy_list = ['남학생A', '남학생B', '남학생C', '남학생D', '남학생E', '남학생F']
girl_list = ['여학생A', '여학생B', '여학생C', '여학생D']
# ------------------------------------

# --- 사용자 입력 받기 (오류 처리 기능 포함) ---
try:
    # 1. 시드값 입력
    seed_input_str = input("대표 학생, 기준 숫자(seed)를 입력하세요: ")
    if seed_input_str is None: raise TypeError("시드 입력이 취소되었습니다.")
    seed_input = int(seed_input_str)

    # 2. 조 인원 입력
    team_size_str = input("한 조에 몇 명씩 배정할까요?: ")
    if team_size_str is None: raise TypeError("조 인원 입력이 취소되었습니다.")
    team_size = int(team_size_str)
    
    # 조 인원은 반드시 1 이상이어야 함
    if team_size <= 0:
        raise ValueError("조 인원은 1명 이상이어야 합니다.")

    # 3. 여학생 수 입력
    girls_per_team_str = input("여학생이 포함된 조에 여학생을 몇 명 배정할까요? (0은 랜덤): ")
    if girls_per_team_str is None: raise TypeError("여학생 수 입력이 취소되었습니다.")
    girls_per_team = int(girls_per_team_str)

# 사용자가 숫자가 아닌 값, 유효하지 않은 값, 또는 입력을 취소했을 경우
except (ValueError, TypeError) as e:
    # 오류(e)를 포함한 안내 메시지를 출력하고, 안전한 기본값으로 프로그램을 계속 실행
    print(f"안내: {e}. 기본값(Seed=42, 조 인원=4, 여학생=랜덤)으로 실행합니다.")
    seed_input = 42
    team_size = 4
    girls_per_team = 0

# --- 알고리즘 실행 ---
# 1. 입력받은 seed 값으로 난수 생성기를 초기화하여 결과의 공정성/재현성 확보
random.seed(seed_input)

# 2. 각 성별 명단을 무작위로 섞음
random.shuffle(boy_list)
random.shuffle(girl_list)

# 3. 전체 학생 명단을 합치고 다시 한번 섞어 무작위성 강화
total_students = boy_list + girl_list
random.shuffle(total_students)

team_count = 1

print("\\n--- 최종 조 추첨 결과 ---")

# 4. 전체 학생 리스트를 '조 인원'만큼씩 잘라서 조를 편성하고 출력
for i in range(0, len(total_students), team_size):
    team = total_students[i:i + team_size]
    print(f"[{team_count}조]: {', '.join(team)}")
    team_count += 1

                    

                         
                    

                    

                

            

        

        
        

            
🛰️ 최종 결과: 보고서 및 평가

            

                [STEP 4] 연구 보고서 살펴보기
                

                    
연구의 모든 과정을 하나의 문서로 정리하기

                    
연구 보고서는 서론, 본론, 결론의 구조를 갖춰 다른 사람이 내 연구 과정을 쉽게 이해하고 신뢰할 수 있도록 작성하는 최종 산출물입니다.

                    
                    
1. 서론

                    
연구 주제: 조건(성비)을 고려한 공정하고 효율적인 조 편성 알고리즘 연구
연구 목적: 수업 시간에 발생하는 조 편성의 비효율과 불만을 코딩을 통해 해결하고자 하였다. 학생 참여(seed 입력)로 공정성을 확보하고, 특정 조건(성비)을 만족하는 자동화 프로그램을 개발하여 그 효율성을 검증하는 것을 목적으로 한다.


                    
2. 연구 수행 내용

                    
이론적 배경 및 선행 연구: 수학적 확률과 파이썬의 의사 난수 생성 원리를 학습하였다. 특히 `random.seed()`를 통한 결과 재현성과 `random.shuffle()`을 이용한 리스트 조작법을 심층적으로 탐구하였다. 선행 연구 조사를 통해 대부분의 조 추첨이 단순 무작위 방식에 의존하고 있어 조건부 추첨에 대한 연구가 필요함을 확인했다.
연구 방법: Python 언어와 `random` 라이브러리를 핵심 도구로 사용하였다. 초기에는 단순 무작위 추출 방식을 구현하였으나, [연구 노트]에서처럼 비효율성을 발견하고, 먼저 명단을 성별로 분리한 뒤 조건에 맞게 조합하는 방식으로 알고리즘을 개선하였다.


                    
3. 연구 결과

                    
단순히 전체 학생을 무작위로 섞는 알고리즘(A)과, 성별을 고려하여 조건에 맞게 조합하는 알고리즘(B)을 모두 구현하였다. 실험 결과, 알고리즘 B는 사용자가 원하는 성비 조건을 항상 만족시키는 결과를 출력함을 확인하였다. 최종적으로 학생 대표가 입력한 seed 값에 따라 항상 동일한 결과가 출력되어, 조작 불가능한 공정한 시스템임을 증명하였다. 개발된 프로그램은 24명 기준 조 편성 시 0.01초 미만의 빠른 실행 속도를 보여 효율성 또한 입증되었다.

                    
                    
4. 결론 및 제언

                    
결론: 본 연구를 통해 파이썬 프로그래밍으로 실생활의 문제를 효과적으로 해결할 수 있음을 확인했다. 특히, `seed` 값을 활용한 공정성 확보와 조건부 알고리즘 설계는 단순한 무작위성을 넘어, 사용자가 원하는 '질서 있는 무작위'를 구현할 수 있다는 점에서 큰 의미가 있다. 코드를 통한 문제 해결 과정에서 논리적 사고력과 문제 해결 능력을 함양할 수 있었다.
제언: 향후 학생들의 '친밀도'나 '학업 성취도'와 같은 더 복잡한 데이터를 변수로 추가하여, 더욱 고도화된 조 편성 알고리즘으로 발전시킬 수 있을 것이다. 또한, 그래픽 사용자 인터페이스(GUI)를 적용하여 사용 편의성을 높이는 방향으로의 개선도 기대된다.

  • 확률 (Probability): 특정 사건이 일어날 가능성을 0과 1 사이의 숫자로 표현한 것.

    • 
                
  • 시간 복잡도 (Time Complexity): 알고리즘의 성능을 나타내는 척도로, 입력값의 증가에 따라 실행 시간이 얼마나 증가하는지를 표현합니다.
