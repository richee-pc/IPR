인구 소멸 위기, 왜 중요할까요?

혹시 '인구 소멸'이라는 말을 들어보셨나요? 이는 특정 지역의 젊은 인구가 어르신 인구에 비해 크게 부족하여, 장기적으로 인구가 줄어들어 지역 공동체가 사라질 위험에 처한 상태를 의미합니다. 이는 단순히 인구가 줄어드는 것을 넘어, 우리 사회의 균형 있는 발전을 위협하는 중요한 문제입니다.

오늘의 프로젝트 목표

이번 시간에는 공공 데이터를 활용하여 우리나라의 인구 소멸 위기 현황을 직접 분석하고, 지역별 소멸 위험도를 계산하여 우리 사회의 모습을 데이터라는 창을 통해 들여다볼 것입니다. 데이터 과학자가 되어 사회 문제를 탐구할 준비, 되셨나요?

  1. KOSIS 접속 및 검색: KOSIS 국가통계포털에 접속 후 '주민등록인구현황'을 검색합니다.
  2. 항목 설정 (가장 중요!): 분석에 필요한 시점, 행정구역, 성별, 연령별 항목을 정확하게 선택합니다.
  3. 데이터 다운로드: '조회' 후 '다운로드' 버튼을 눌러 **'XLSX'** 형식으로 저장합니다. (파일명: `population_data.xlsx`)

STEP 1: 데이터 불러오기와 첫인상 확인

무엇을 하는 코드인가요?

우리가 다운로드한 엑셀 파일을 파이썬으로 불러와서, 앞으로 데이터를 분석할 작업대인 '데이터프레임' 위에 올려놓는 과정입니다.

코드 실행

# 필요한 라이브러리를 불러옵니다.
import pandas as pd
import numpy as np

# 다운로드한 엑셀 파일을 'df'라는 작업대에 올립니다.
df = pd.read_excel('population_data.xlsx')

# 데이터가 어떻게 생겼는지 앞부분 5줄만 살펴봅니다.
df.head()
🔎 코드 해설
  • import pandas as pd: 'pandas' 라이브러리를 앞으로 'pd'라는 짧은 별명으로 부르겠다고 약속하는 것입니다.
  • pd.read_excel(...): 엑셀 파일을 읽어서 **데이터프레임(DataFrame)** 이라는 표 형태로 만들어주는 함수입니다.
  • df.head(): 데이터프레임의 첫 5줄을 보여주는 함수로, 데이터가 잘 불러와졌는지, 어떤 열들이 있는지 빠르게 확인할 때 사용합니다.
[시각 자료 예시: 엑셀 시트가 판다스 데이터프레임(행과 열 이름이 있는 표)으로 변환되는 모습을 보여주는 이미지]
STEP 2: 데이터 다듬기 (열 이름 변경 및 결측값 처리)

무엇을 하는 코드인가요?

분석에 방해가 되는 불필요한 행을 제거하고, 복잡한 열 이름을 간결하게 바꾸며, 비어있는 칸(결측값)을 채워 넣어 데이터를 깨끗하게 청소하는 과정입니다.

코드 실행

# 불필요한 첫 행을 제거합니다.
df.drop(0, inplace=True)

# 복잡한 열 이름을 간결하게 바꿉니다.
df.rename(columns={'행정구역(시군구)별': '시군구', '총인구수 (명)': '총인구수'}, inplace=True)

# 비어있는 값(결측값)을 0으로 채웁니다.
df.fillna(0, inplace=True)

# 처리 후 데이터 상태 확인
df.head()
🔎 코드 해설
  • df.drop(0, inplace=True): 0번째 행을 제거합니다. inplace=True 옵션은 '변경사항을 바로 저장해줘!'라는 의미로, 원본 데이터를 바로 수정합니다.
  • df.rename(columns={...}): 딕셔너리 형태로 {'기존이름':'새이름'}을 지정하여 열 이름을 바꿉니다.
  • df.fillna(0, inplace=True): 데이터의 모든 빈칸(NaN)을 0으로 채워 넣어 계산 오류를 방지합니다.
STEP 3: 새로운 분석용 열 만들기

무엇을 하는 코드인가요?

여러 연령대별로 나뉘어 있는 인구 데이터를, 우리의 분석 목표에 맞게 '20-39세' 인구와 '65세 이상' 인구라는 두 개의 새로운 그룹으로 합쳐주는 과정입니다.

코드 실행

# 먼저, 연령별 인구 데이터가 숫자가 아닌 문자열일 수 있으므로 숫자로 변환합니다.
age_cols = ['20-24세', '25-29세', '30-34세', '35-39세', '65-69세', '70-74세'] # ... 등 모든 연령 열
for col in age_cols:
    df[col] = pd.to_numeric(df[col], errors='coerce').fillna(0)

# 20-39세 인구와 65세 이상 인구를 계산하여 새 열에 저장합니다.
df['20-39세'] = df['20-24세'] + df['25-29세'] + df['30-34세'] + df['35-39세']
df['65세이상'] = df['65-69세'] + df['70-74세'] # ... 모든 65세 이상 연령대 더하기

# 새로 만든 열 확인
df[['시군구', '20-39세', '65세이상']].head()
🔎 코드 해설
  • pd.to_numeric(...): 데이터의 형식을 숫자로 바꿔주는 함수입니다. errors='coerce'는 숫자로 바꿀 수 없는 값이 있다면 강제로 결측값(NaN)으로 만들라는 의미입니다.
  • df['새로운열이름'] = df['기존열1'] + df['기존열2']: 기존 열들을 이용한 간단한 사칙연산으로 새로운 열을 쉽게 만들 수 있습니다.
STEP 4: '소멸 위험 지수' 계산하기

무엇을 하는 코드인가요?

이 프로젝트의 핵심! '20-39세 인구'를 '65세 이상 인구'로 나누어 '소멸위험지수'를 계산합니다. 이 지표를 통해 각 지역이 얼마나 활기찬지, 혹은 소멸 위기에 처해있는지를 객관적으로 파악할 수 있습니다.

코드 실행

# 분석에 필요한 열만 선택하여 새로운 작업대(analysis_df)를 만듭니다.
analysis_df = df[['시군구', '20-39세', '65세이상']].copy()
analysis_df.set_index('시군구', inplace=True)

# '65세이상' 인구가 0인 경우 발생하는 오류를 방지합니다.
analysis_df['65세이상'].replace(0, 1, inplace=True)

# 소멸 위험 지수 계산! (핵심 공식)
analysis_df['소멸위험지수'] = analysis_df['20-39세'] / analysis_df['65세이상']

# 계산 결과를 지수가 낮은 순으로 정렬하여 확인합니다.
analysis_df.sort_values(by='소멸위험지수').head(10)
🔎 코드 해설
  • .copy(): 원본 시험지를 그대로 두고, 복사본에 문제를 푸는 것과 같아요. 원본 데이터를 안전하게 지킬 수 있습니다.
  • .set_index('시군구'): '시군구' 열을 데이터의 대표 이름표(인덱스)로 지정하여 데이터를 더 쉽게 찾을 수 있게 합니다.
  • .sort_values(by='소멸위험지수'): '소멸위험지수' 열을 기준으로 데이터를 오름차순으로 정렬하여 어떤 지역이 가장 위험한지 쉽게 파악할 수 있습니다.
STEP 5: 조건에 따라 데이터 분류하기 (Numpy.where)

무엇을 하는 코드인가요?

계산된 소멸위험지수를 바탕으로, 각 지역의 위험 등급을 '소멸고위험', '소멸주의', '소멸저위험' 세 그룹으로 나누어주는 작업입니다. 복잡한 데이터를 단순한 카테고리로 만들어 한눈에 파악하기 쉽게 해줍니다.

코드 실행

# 소멸 위험 지수(ext)에 따라 지역의 상태를 분류하여 '소멸위험등급' 열을 만듭니다.
ext = analysis_df['소멸위험지수']
analysis_df['소멸위험등급'] = np.where(ext < 0.5, '소멸고위험', 
                                  np.where(ext < 1.0, '소멸주의', '소멸저위험'))

# 등급별 지역이 몇 개씩 있는지 확인합니다.
analysis_df['소멸위험등급'].value_counts()
🔎 코드 해설
  • np.where(조건, 참일때, 거짓일때): 엑셀의 IF 함수처럼, 조건을 만족하면 '참일때' 값을, 아니면 '거짓일때' 값을 부여합니다. 이 코드는 IF 함수를 중첩해서 사용한 것과 같습니다.
  • .value_counts(): 특정 열에 있는 값들이 각각 몇 개씩 있는지 세어주는 편리한 함수입니다.
[시각 자료 예시: np.where 함수가 소멸위험지수라는 하나의 기준선을 따라 데이터를 '고위험', '주의', '저위험' 세 그룹으로 나누는 과정을 보여주는 순서도 이미지]
